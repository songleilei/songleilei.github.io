react-Fiber

0001-01-01 477 words 1 min read

我们在 React 中做的就是践行代数效应（Algebraic Effects）

代数效应与 Fiber 架构的联系

代数效应是函数式编程中的概念，用于将副作用从函数调用中分离

Fiber(纤程)

进程（Process） 线程（Thread） 协程（Coroutine） Generator

Fiber 含义

  1. 老的 React 架构 数据保存在调用栈中，被称为 Stack Reconciler 子节点用 child 参数连接，父节点用 return 连接。 老的架构中使用递归的形式工作 新结构中采用遍历的形式实现可中断的递归，也复用了这种形式

  2. 新的 React 架构 基于 Fiber 节点实现的，被称为 Fiber Reconciler。

    1. 作为静态数据结构，每个 Fiber 节点对应一个组件，保存了该组件的类型，对应的 dom 节点等信息，即虚拟 dom。

    2. 作为动态的工作单元

什么是双缓存

内存中构建并直接替换的工作称为双缓存

当首次调用 ReactDom.render 时，会创建整个应用的根节点 FiberRootNode， 每次调用 ReactDom.render 会创建当前应用的根节点 RootFiber，首屏渲染之前 RootFiber 没有子节点。 接下来进入首屏渲染的逻辑，不管是 setState 或者 useState 更新，都会从根节点创建 RootFiber，深度优先遍历模拟递归，最终形成 current Fiber 树 和 workInProgress Fiber 树
